# PR 説明テンプレート集

`pr-description` skill から参照される、PR 種別ごとのテンプレート。

## 汎用テンプレート(機能追加・改修向け)

```markdown
## 概要 / Summary
1〜2文で「何を、なぜ」変更したか

## 背景 / Context
- なぜこの変更が必要か
- 関連 Issue: #123
- 関連ドキュメント: <link>

## 変更内容 / Changes
- [機能] ユーザー登録時のメール確認フローを追加
- [機能] `/auth/verify` エンドポイント新設
- [改修] 既存の `/auth/register` レスポンスに `verification_required` フィールドを追加
- [テスト] ユニットテスト 12件、E2Eシナリオ 3件追加

## スクリーンショット / Screenshots
| Before | After |
| :--- | :--- |
| (UI変更がある場合) | |

## 動作確認 / How to Test
1. `pnpm install` 後 `pnpm dev`
2. `/register` でアカウント作成
3. 受信メールのリンクを踏み、ダッシュボードに遷移することを確認

## 影響範囲 / Impact
- **ユーザー影響**: 新規登録フローが変わる(既存ユーザーは影響なし)
- **DB**: マイグレーション 1件追加(後方互換あり)
- **環境変数**: `MAIL_FROM_ADDRESS` 追加が必要
- **API 互換性**: 既存クライアントは動作継続(新フィールドは optional)

## レビュー観点 / Review Notes
- ⚠️ メール送信のエラー時の挙動を特に見てほしい(`services/mail.ts:80`)
- 暗号化方式の選定理由は `docs/decisions/0012.md` 参照

## チェックリスト
- [ ] テスト追加・更新済み
- [ ] ドキュメント更新済み(README/API spec)
- [ ] CHANGELOG 更新済み
- [ ] 環境変数の追加を共有済み
- [ ] 破壊的変更なし(または移行手順を記載)
- [ ] セルフレビュー実施済み

Closes: #123
```

## Bugfix PR

```markdown
## 概要
〜のバグを修正。

## 再現手順
1. ...
2. ...

## 原因
〜が〜の条件で〜してしまっていた。

## 修正方針
〜することで対応。代替案として〜も検討したが、〜の理由で採用しなかった。

## 影響範囲・回帰リスク
- 影響: ...
- 追加テスト: 該当ケースの回帰テストを `xxx.test.ts` に追加

Fixes: #234
```

## Refactor PR

```markdown
## 概要
挙動を変えずに〜を整理。

## 動機
- 〜が分散していて変更コストが高い
- 今後の機能追加(#300)に備える前準備

## 変更内容
- ...
- ⚠️ 公開API/外部仕様は変更していない

## 挙動が変わっていないことの確認
- 既存テスト全件パス
- カバレッジ低下なし(80% → 81%)
- 主要画面のスモークテスト実施
```
